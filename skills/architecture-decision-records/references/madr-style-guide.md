# MADR Style Guide

## Lean writing rules

- keep the ADR small enough to scan quickly
- write in decisive language
- separate context from decision from consequences
- prefer real alternatives over strawman options
- include both positive and negative consequences

## Status discipline

Use status consistently. When a new ADR replaces an old one, say so directly instead of leaving both records to contradict each other.

## Tone

The document should explain the decision to a future engineer, not relive the full discussion transcript.
