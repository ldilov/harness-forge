# Get Started Guides

Harness Forge now exposes role-oriented entrypoints so operators can start with
the guidance that matches their job instead of navigating the full repo map up
front.

## Role Guides

- [Adopter](./adopter.md): install minimal, recommended, or full runtime packs and inspect what landed.
- [Maintainer](./maintainer.md): validate releases, review runtime truth surfaces, and keep catalogs coherent.
- [Pack author](./pack-author.md): add or extend pack definitions without breaking provenance or support posture.
- [Troubleshooting](./troubleshooting.md): explain managed surfaces, inspect drift, and choose safe recovery commands.

## Start Here

- New to Harness Forge: read [Adopter](./adopter.md)
- Shipping or releasing Harness Forge: read [Maintainer](./maintainer.md)
- Extending the platform or authored content: read [Pack author](./pack-author.md)
- Debugging a workspace or update: read [Troubleshooting](./troubleshooting.md)
